# 伊東園ホテル 空室監視ツール

伊東園ホテルの空室が出たら、スマホに通知を飛ばすツールです。
**予約は自動化しません**。「空きが出たことに気づく」だけを自動化し、
実際の予約操作は自分の手で行う設計です（理由は下記「なぜ自動予約にしないか」参照）。

## できること

- 指定した宿泊日の空室状況を定期的にチェック
- 「満室 → 空きあり」に変わった瞬間だけ通知（毎回通知は来ません）
- 通知はntfy.sh（無料・登録不要）かDiscord Webhookから選べます

## セットアップ

### 1. 必要なものをインストール

```bash
pip install playwright
playwright install chromium
```

### 2. 通知先を用意する

**Gmailを使う場合（おすすめ・今回の設定）**
1. 送信元にするGoogleアカウントで2段階認証を有効にする
   （すでに使っているアカウントで良いが、心配なら送信専用に
   新しいGoogleアカウントを1つ作ってもいい）
2. https://myaccount.google.com/apppasswords にアクセスし、
   アプリパスワードを発行する（「アプリを選択」で「メール」、
   「デバイスを選択」で「その他」を選び、名前は何でもいい）
3. 発行された16桁のパスワード（スペース入りでOK）を
   `watcher.py` の `GMAIL_APP_PASSWORD` に貼る
4. `GMAIL_ADDRESS` に送信元アカウントのアドレス、
   `EMAIL_TO` に通知を受け取りたいアドレス
   （今回は `zakusenwig@gmail.com`）を設定する
5. `NOTIFY_METHOD = "email"` になっていることを確認

普段使っているメインのGmailアカウントの通常ログインパスワードは
`GMAIL_APP_PASSWORD` には使えません（Googleの仕様で、2段階認証を
有効にしたアカウントは専用の「アプリパスワード」を発行する必要が
あります）。

**ntfy.shを使う場合（スマホにアプリを入れる方式）**
1. スマホに「ntfy」アプリを入れる（iOS/Android両方あります）
2. アプリ内で好きなトピック名を購読する（他人と被らない適当な文字列。
   例: `itoen-watch-7x9k2z`）
3. `watcher.py` の `NTFY_TOPIC` にその名前を設定し、
   `NOTIFY_METHOD = "ntfy"` に変更する

**Discordを使う場合**
1. 通知を受け取りたいチャンネルの設定から「連携サービス」→「Webhookを作成」
2. 発行されたURLを `watcher.py` の `DISCORD_WEBHOOK_URL` に設定
3. `NOTIFY_METHOD = "discord"` に変更

**参考：LINEについて**
LINE Notifyは2025年3月末にサービス終了しています。後継の
「LINE Messaging API」は、BOTアカウントの作成・チャネル発行など
セットアップの手間が大きく、今回のような個人利用には見合わないため
対応していません。

### 3. 監視したいホテル・日付・人数構成を設定する

`watcher.py` 内を編集：

```python
SEARCH_URL = "https://www5.489pro.com/asp/g/c/calendar.asp?kp=itoen&ty=&sp=&lan=JPN"

TARGET_HOTELS = [
    "伊東園ホテル",
    "伊東園ホテル別館",
    "伊東園ホテル松川館",
]

TARGET_DATE = "2026-07-25"

ADULTS_PER_ROOM = 1   # 1部屋あたりの大人人数
NIGHTS = 1
ROOMS = 1             # 大人1名・1部屋で検索
```

「エリアまたは施設」のプルダウンは1回の検索で1施設しか選べないため、
このスクリプトは指定した3施設分を**1回の実行の中で順番に検索**します
（3施設×1回のアクセス、という挙動です）。

### 4. 一度手動で動かして、セレクタを実物に合わせる

```bash
python watcher.py
```

伊東園ホテルの予約ページはJavaScriptで動くフォームなので、
`find_availability()` 内の「日付選択」「検索ボタン」「カレンダーの記号」を
拾う部分は、**実際のページ構造に合わせた微調整が必要になる可能性が高いです**。

調整のコツ：
1. `headless=True` を一時的に `headless=False` に変えて、
   実際にブラウザが動く様子を目で確認する
2. `screenshots/` フォルダに毎回自動で結果ページのスクリーンショットが
   保存されます。プログラムの自動判定を信用しきる前に、しばらくは
   このスクリーンショットも目視で突き合わせて、誤判定がないか
   確認してください
3. ブラウザの開発者ツール（右クリック→検証）で、日付選択欄や
   検索ボタンの実際の `name` / `id` / テキストを確認し、
   スクリプト内のセレクタを書き換える

### 5. 定期実行する

2つのやり方があります。用途に応じて選んでください。

**方式A: スクリプト自体をループさせる（お手軽・PCをつけっぱなしにする前提）**

```cmd
python watcher.py --loop
```

`CHECK_INTERVAL_MINUTES`（デフォルト20分）ごとに自動でチェックを繰り返します。
止めるときはウィンドウで`Ctrl+C`。ウィンドウを閉じたり、PCがスリープ/再起動
すると止まります。ちょっと試したいときや、旅行直前の数日だけ集中して
見張りたいときに向いています。

**方式B: タスクスケジューラで定期的に1回ずつ実行（安定運用向け）**

「タスクの作成」→トリガーを「20分ごとに繰り返す」に設定し、
アクションに（`--loop`を付けずに）`python.exe watcher.py`を指定します。
PCの再起動やスリープからの復帰にも強く、長期間放置するならこちらが安定です。

**方法C: cron（Mac/Linux）**

```bash
crontab -e
```

以下を追加（20分おきに実行する例）：

```
*/20 * * * * cd /path/to/itoen_watcher && /usr/bin/python3 watcher.py >> watcher.log 2>&1
```

**方法D: GitHub Actions（おすすめ・自分のPCの電源やネット環境に一切依存しない）**

GitHubというコード置き場のサービスには、決まったスケジュールで
コードを自動実行してくれる「Actions」という無料機能があります。
これを使うと、あなたのPCがオフでも、24時間365日勝手にチェックが
続きます（`.github/workflows/watch.yml`は同梱済みです）。

手順：

1. https://github.com でアカウントを作る（未所持の場合）
2. 新しいリポジトリを作成する（`itoen-watcher`など、名前は何でもいい。
   **Private（非公開）を選んでください**）
3. この`itoen_watcher`フォルダの中身一式（`watcher.py`、`README.md`、
   `.github/workflows/watch.yml`）をそのリポジトリにアップロードする
   （GitHubのWeb画面の「Add file」→「Upload files」で、フォルダごと
   ドラッグ＆ドロップでOK。gitコマンドに慣れていなくても大丈夫です）
4. リポジトリの「Settings」→「Secrets and variables」→「Actions」を開き、
   「New repository secret」から以下の3つを登録する：
   - `GMAIL_ADDRESS`：送信元Gmailアドレス
   - `GMAIL_APP_PASSWORD`：発行したアプリパスワード
   - `EMAIL_TO`：`zakusenwig@gmail.com`
5. リポジトリの「Actions」タブを開くと、ワークフローが表示されます。
   有効化されていなければ「I understand my workflows, go ahead and
   enable them」をクリック
6. あとは自動的に30分おきに実行されます。すぐ試したければ「Actions」
   タブ→該当ワークフロー→「Run workflow」ボタンで手動実行もできます

**GitHub Actionsの無料枠について**
- Privateリポジトリは月2,000分まで無料（Publicリポジトリなら無制限）
- 1回の実行が約1〜2分として、30分おき（1日48回）なら月あたり
  1,500〜3,000分程度。ギリギリ〜やや超える可能性があるので、
  枠が心配なら`watch.yml`内の`cron`を`*/30`→`*/60`（1時間おき）に
  緩めるか、リポジトリをPublicにしてください
  （コード自体に秘密情報は含まれていないので、Publicにしても
  パスワードが漏れることはありません。Secretsは別管理です）
- スケジュール実行は数分程度前後にずれることがあります
  （GitHub側の混雑状況次第）。「正確に20分おき」までの厳密さは
  期待しないでください

**注意点**
- `state.json`（前回結果の記憶）は実行のたびにリポジトリへ
  自動コミットされます。差分だけの小さなコミットなので、
  リポジトリが大きく膨らむことはありません
- スクリーンショットはリポジトリにコミットせず、Actionsの
  「アーティファクト」として7日間だけ保存されます
  （Actions実行結果のページからダウンロードできます）

## なぜ自動予約にしないか

技術的には「空きを見つけたら自動でフォーム送信して予約確定まで」
やることも不可能ではありませんが、あえてやっていません。

- 489pro（このホテル予約システム）の利用規約は、
  人がブラウザで操作することを前提にしており、
  bot経由の自動予約は想定されていません。見つかった場合、
  最悪予約自体が無効化されるリスクがあります
- カード情報などの決済処理を個人スクリプトに持たせるのは、
  開発コストの割に得られるものが小さく、セキュリティ的にも
  見合いません
- 「通知が来た瞬間に自分で予約確定する」で、リスクはほぼゼロのまま
  実用上のスピードはほとんど落ちません

## 注意事項

- アクセス間隔は15〜30分に1回程度を目安にしてください。
  短すぎるとサーバーに負荷をかけ、アクセス制限を受ける可能性があります
- サイトの構造は予告なく変わることがあります。動かなくなったら
  まず `headless=False` にして実際の画面を確認してください
- このツールはあくまで補助であり、空室情報の正確性・取得の確実性を
  保証するものではありません。最終判断は必ず公式サイトで確認してください
